OnStream main pages

OnStream Linux SCSI tape driver development

This page coordinates the development of a Linux tape driver for the OnStream SC-30/SC-50 devices. The target is to create a kernel driver to provide an st-like interface to the userspace. The driver will be put under GPL and submitted to the kernel folks for inclusion into the mainstream kernel.

This project is a joint effort of indiviuals that try to get their OnStream SC-x0 ADR SCSI tape drive to run with Linux. They are supported by OnStream and SuSE. This machine is dedicated for the support of this and is located in Eindhoven, NL. It should be reachable via the IP address 195.193.200.24 and -- in a few days -- via the name linux1.onstream.nl.

This web page is a trial to coordinate this effort. Besides from pointers to relevant information or software, it should also give a short summary of the tools for the developers which are used. It does not provide software for the end user! As soon as an alpha version is out there, it will be put at some location in order to allow people who want to take the risk to play with it. As soon as there is a beta, it will be announced here and some download page will be installed.

CVS

For the joint development, a CVS repository is installed on this machine. It handles the driver sources and also the web pages.

The repository contains the modules (=directories) WebPages, Driver, Docu, Firmware and Misc.

In Driver you find the stock 2.2.14 kernel st driver and some support files like include headers. A branch with the tag L23 is created for the Linux-2.3 versions of the stdriver. That way, changes to the 2.2 version should be easily transferable to the 2.3 branch and vice versa. Here lives the kernel tape driver for OnStream tapes osst. Of course, there can be added new files like scripts to test or TODO lists ...

The Docu directory contains the docu from OnStream, namely the ADR spec as a PDF. More documentation will go here. The Misc directory contains the IDE driver for the OnStream of Gadi Oxman, and the userspace OnStream SCSI driver from Terry Hardie. Other sourcecode related to the OnStream should also go here.

The Firmware directory contains DOS and Wind*ws programs to update the firmware of your OnStream tape device.

CVS read access is granted upon request. If you have some small contributions, the diffs (use diff -u) should be send to the mailing list and someone will have a look and check it into the CVS. If you are contributing a lot, it will be easier to give you read-write access to the repository, so this will happen.

Mailing list

A mailing list is there to provide an easy way for the developers to communicate. The name is osst@linux1.onstream.nl, and it's a majordomo list. So subscribe requests etc. go to osst-request.

Related efforts and resources

Terry Hardie has created a userspace driver, which uses the scsi generic (sg) of Linux to access the device. It can be downloaded.

There is a IDE driver for the OnStream DI-30, which uses the same format for the tapes defined by the ADR soec. It was written by Gadi Oxman and is most probably going to be integrated into the next kernels.

SCSI specs are unfortunately not freely available. Some copies of the draft version of the SCSI-II spec are floating around. Drafts for the upcoming SCSI-III have also been seen ...
Actually, you can find the draft versions on the pages of the T10 committee. I don't know, why the ANSI does not allow the final approved versions to be freely available. Shouldn't the bad VESA example tell them, they are wrong? Free software is in trouble if standards are hidden.

Status
2000-02-22
CVS is up and running, and almost completely filled with the base data. For the mailing list, the DNS entry needs to be there, first.
2000-03-03
While the userspace osg gets better and better, the kernelspace OnStream SCSI tape driver osst starts to become useable. Reading and writing basically works, but the error recovery needs to be implemented before an alpha release can be published.
2000-03-30
Mailing list is up and running without any problems since some time now. Subscribe by sending a mail with help in the body to osst-request@linux1.onstream.nl and follow the instructions.
2000-04-06
A tester's page has been set up, where you can find a tarball with the driver (ALPHA state at the moment) to download and some instructions on how to use it and how to report success or failure back to the developers.
2000-05-05
The Firmware dir has been added to CVS
2000-12-15
Alan Cox merged the osst driver 0.8.6 into Linux-2.2.19pre1.
Hopefully Linus will accept to merge the osst - 0.9.4 (which is functionally equivalent) into 2.4.0.
2000-12-20
Added notice that the Tester's page is the one being interesting ...
The osst driver by now is very stable -- thanks to the beta testers -- and should be prefered over osg. It even works with the IDE device (DI-30) over ide-scsi and for the USB (USB30) one via usb-storage. For the latter, you need at least a 2.4.0-test12 kernel and you need to activate freecom support.
